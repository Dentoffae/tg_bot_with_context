# TG Bot With Context — Отчёт по прогонам

Telegram-бот на aiogram 3 + OpenAI Chat Completions API с контекстом диалога в памяти.

---

## Структура проекта

```
TG_BOT_WITH_CONTEXT/
├── bot.py               # aiogram-бот, обработчики сообщений
├── config.py            # настройки: токены, модель, параметры генерации
├── openai_client.py     # асинхронный клиент OpenAI
├── context_manager.py   # хранение истории диалога (dict в RAM)
├── .env                 # BOT_TOKEN, OPENAI_API_KEY, PROXY_URL
└── requirements.txt     # зависимости
```

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Приветствие + сброс контекста |
| `/clear` | Очистить историю диалога |
| `/info` | Показать количество сообщений в контексте |
| `очистить контекст` | То же, что `/clear`, текстом |

## Выводы

### gpt-5-2025-08-07 — проблемы при тестировании

| Проблема | Причина |
|---|---|
| `max_tokens` → ошибка 400 | Параметр переименован в `max_completion_tokens` |
| `temperature=0.7` → ошибка 400 | Модель поддерживает только `temperature=1` (default) |
| `finish_reason=length`, пустой контент | Модель — reasoning-типа: тратит токены на внутренние рассуждения, при лимите 1024 не успевает сгенерировать видимый ответ |
| Зависание до 37 сек. | Долгий reasoning перед генерацией |

**Решение:** модель переключена на `gpt-4o-mini`. Для корректной работы `gpt-5-2025-08-07` нужно поднять `max_completion_tokens` до 8000–16000+.

### gpt-4o-mini — стабильная работа

- Все 4 прогона: `finish_reason=stop` ✅
- Латентность: 1.6–7.4 сек (первый запрос холоднее из-за TLS-handshake)
- Ответы на русском, структурированные, адекватные
- Суммарные расходы за 4 обмена: **≈ $0.00047** (меньше 0.05 копейки)

### Влияние temperature (gpt-4o-mini)

| temperature | Характер ответов |
|---|---|
| 0.0–0.3 | Детерминированные, сухие, точные |
| **0.7** (текущий) | Сбалансированный: информативно + лёгкий стиль |
| 0.9–1.0 | Более творческие формулировки, иногда с отступлениями |
